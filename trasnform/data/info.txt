Для получения данных в нужном формате необходимы следующие входные данные:
- файл с информацией о детских садах (пример - УДО.json)
- файл с информацией о других учебных заведениях (пример - УОСО.json)

Далее запускаем файл transform.bat
Файл содержит строку вида:
java -jar trasnform.jar "название файла c детскими садами" "название файла с другими УО" "папка с файлами (полный путь)" "название файла с конечными данными"
Все 4 параметра не являются обязательными. Если они не указаны, то будут подставлены значения по умолчанию:
название файла c детскими садами - УДО.json
название файла с другими УО - УОСО.json
папка с файлами - текущая (та, из которой запускается файл transform.bat и там же должен лежать файл trasnform.jar)
название файла с конечными данными - data.json
В этом случае строка в transform.bat будет иметь вид:
java -jar trasnform.jar
ВАЖНО!!! Должен соблюдаться полрядок указания параметров. Это значит, что все ПРЕДЫДУЩИЕ параметры должны присутствовать в строке запуска.
Но если нам их указывать не обязательно, можно в качестве значения указать пустую строку.
Например, нам нужно указать только имя файла с другими УО - УОСО2.json. Тогда строка будет иметь вид:
java -jar trasnform.jar "" "УОСО2.json"

Если в указанной папке уже существует файл с выходыми данными с указанным нами названием,
то в процессе нового запуска transform.bat этот файл будет пересоздан заново с новыми данными.
Старые данные при этом будут затерты безвозвратно.

В течение работы данные о процессе выводятся как в консоль, так и в лог-файл logfile.log.
При нормальной штатной работе утилиты (когда данные в файлах распознаны и найдены целиком по всем районам, областям и т.д.), в логе будут всего 2 строки
LogFile is created!
DataFile is created! data.json
